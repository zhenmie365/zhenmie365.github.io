firefox证书导入问题

Posted on 2018-06-13 | In working , web , browser | Visitors:

概述

firefox在58.0版本后就默认用cert9.db做一些数据管理，例如导入证书的存储。
而用NSS certutil编程式地导入证书就会出现不成功的案例。

shared db

firefox有几种db类型：

certutil supports two types of databases: the legacy security databases (cert8.db, key3.db, and secmod.db) and new SQLite databases (cert9.db, key4.db, and pkcs11.txt).

NSS recognizes the following prefixes:
sql: requests the newer database
dbm: requests the legacy database

在linux上，在~/.bashrc设置全局环境变量
export NSS_DEFAULT_DB_TYPE=”sql”
或在certutil命令中在-d指定的db路径参数后加前缀“sql:”就可以修改相对应的db。如：
certutil.exe -A -t “trust_type” -i “cert_file” -d “sql:profile_path”

问题就在于windows系统，db路径前加前缀”sql:”是行不通，会抛出：
certutil: NSS_Initialize failed: security library: bad database
一系列错误，其本质都是说制定的db是不合法。
有网友话是NSS certutil的版本不够高，所以我试了NSS3.8和NSS3.9都会出现同样错误。
在NSS的版本库里看到最新的NSS都是15年出的，都感觉不够新，所以判断在windows系统里，NSS不支持操作cert9.db这类数据库。
至于怎样在windows系统编程式的操作shared cert9.db，现在还没有方法。
就如近期有网友说“However much sad enugh that mozilla doesnt care about this.”
连mozilla都不care啦，我们开发人员唯有说“此功能不支持了！”
